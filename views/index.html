JSON Data Input Format

Only JSON Data input is recognized. Multipart Form input or Urlencoded input are not recognized.

 
{  
    "name": "owner name",  
    "email": "owner email",  
    "country": "owner country"  
}

JSON Data Output Format

{  
    "message": "success" || "error message",  
    "data": result data object || "null" if error  
}

User Routes

  • Can view the Readme of the API (/) GET
  • Can create an entry (/api/data/create) POST
  • Can read all entries (/api/data) GET
  • Can read an entry with id (/api/data/:id) GET
  • Can update an entry with id (api/data/:id/update) PUT
  • Can delete an entry with id (api/data/:id/delete) DELETE

Data Model

  • Name (required)
  • Email (required)
  • Country (required)
